# خطة التحكيم البشري وقياس الجودة

## الهدف

تحويل قاعدة الفرز الحالية إلى مجموعة بيانات محكّمة يمكن استخدامها في البحث وبناء طبقة موثوقة للعلاقات عبر دورات المراجعة.

## المرحلة الأولى: تثبيت دليل الترميز

- تعريف العيب المؤثر في قابلية الموافقة.
- تحديد حدود Additional Comments.
- تثبيت قواعد دمج وتقسيم العيوب الذرية.
- تحديد الفرق بين المشكلة والإجراء العلاجي.
- تجميد إصدار `Coding Manual v1.0` مع سجل تغييرات.

## المرحلة الثانية: التحكيم المزدوج

يقوم مراجعان مستقلان بتقييم عينة طبقية تشمل:

- مجالات تنظيمية مختلفة.
- طلبات من دورتين ومن ثلاث دورات فأكثر.
- نصوصًا نظيفة ومحجوبة ومتأثرة بـOCR.
- رسائل لم تنتج مرشح A.
- روابط ذات تشابه مرتفع ومنخفض.

## محاور التحكيم

1. هل المقطع عيب حقيقي؟
2. هل يؤثر في قابلية الموافقة؟
3. هل حدود المقطع صحيحة؟
4. هل يمثل عيبًا ذريًا واحدًا؟
5. هل المجال ورمز السبب صحيحان؟
6. هل الإجراء مرتبط بالعيب المناسب؟
7. ما نوع العلاقة عبر الدورة؟

## مقاييس الجودة

- Precision وRecall وF1 على عينة ممثلة.
- Cohen's kappa لاتفاق المراجعين.
- فواصل ثقة تراعي تجمّع المقاطع داخل الخطابات.
- مقاييس مستقلة على مستوى الرسالة والمقطع والعيب الذري.

## أنواع العلاقات النهائية المقترحة

- Exact persistence
- Semantic persistence
- Partial resolution
- Escalation
- Same dependency, different failure mode
- Deficiency migration
- New deficiency
- No valid relationship
- Unassessable
